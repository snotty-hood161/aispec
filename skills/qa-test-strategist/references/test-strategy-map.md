# 测试场景 → 测试策略映射表

用此表将功能场景映射到对应的测试类型、优先级和覆盖要求。

## 使用方式
1. 从 PRD 和 Spec 中提取功能场景。
2. 按下表确定每个场景的测试类型和优先级。
3. P0 场景必须编写测试用例，P1 建议编写，P2 按资源情况决定。

---

## 场景路由表

| 编号 | 功能场景 | 测试类型 | 优先级 | 覆盖要求 |
|------|---------|---------|--------|---------|
| A | 用户注册/登录/登出 | E2E + 集成 | P0 | 正向 + 异常（错误密码、重复注册） |
| B | 密码重置/找回 | E2E | P0 | 正向 + Token 过期 |
| C | 权限控制（不同角色的可见性） | E2E + 集成 | P0 | 每个角色至少 1 个正向 + 1 个越权 |
| D | CRUD 核心业务对象 | E2E + 单元 | P0 | 创建/读取/更新/删除各 1 个正向 |
| E | 列表查询（搜索/筛选/分页/排序） | E2E + 单元 | P1 | 关键筛选条件 + 空数据 + 大数据量 |
| F | 表单提交（校验/提交/反馈） | E2E | P1 | 必填校验 + 格式校验 + 边界值 |
| G | 文件上传/下载 | 集成 + E2E | P1 | 格式限制 + 大小限制 + 类型伪造 |
| H | 支付/退款流程 | E2E + 集成 | P0 | 正向支付 + 余额不足 + 超时 + 退款 |
| I | 消息/通知推送 | 集成 | P1 | 发送成功 + 模板渲染 + 频率限制 |
| J | 定时任务/后台作业 | 集成 + 单元 | P1 | 执行成功 + 幂等性 + 异常恢复 |
| K | 第三方集成（支付/短信/OSS） | 集成（Mock） | P1 | 正向 + 超时 + 降级 |
| L | 数据导入/导出 | 集成 | P1 | 大文件 + 格式错误 + 编码问题 |
| M | 多租户隔离 | 集成 + E2E | P0 | 租户 A 不可见租户 B 数据 |
| N | 并发操作（乐观锁/幂等） | 集成 | P1 | 同时修改同一资源 |
| O | 缓存一致性 | 集成 | P2 | 写入后读取一致 + 缓存失效 |
| P | API 版本兼容 | 集成 | P2 | 旧客户端兼容新 API |
| Q | 性能基线 | 性能 | P1 | 核心 API P95 < SLA |
| R | 安全测试 | 安全 | P0 | 注入 + XSS + CSRF + 越权 |
