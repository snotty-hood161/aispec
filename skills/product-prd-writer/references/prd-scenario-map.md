# PRD 编写阶段 → 工作内容映射表

用此表引导 PRD 编写的五个阶段和每个阶段的工作内容。

## 使用方式
1. 按阶段顺序逐步执行。
2. 每个阶段结束后确认产出，再进入下一阶段。
3. 已有部分输入时可跳过对应阶段（如已有竞品报告则跳过 Phase 2）。

---

## 阶段路由表

| 阶段 | 名称 | 引导问题文件 | 核心产出 | MCP 依赖 |
|------|------|------------|---------|---------|
| Phase 1 | 市场调研 | `agents/product/phases/01-research.md` | 行业概况、目标市场、关键洞察 | 搜索 MCP（必需） |
| Phase 2 | 竞品分析 | `agents/product/phases/02-competitive.md` | 竞品矩阵、SWOT、差异化定位 | 搜索 MCP + Playwright（可选） |
| Phase 3 | 需求定义 | `agents/product/phases/03-requirements.md` | 用户画像、用户故事、优先级矩阵 | 无 |
| Phase 4 | MVP 路线图 | `agents/product/phases/04-roadmap.md` | MVP 功能集、成功指标、迭代计划 | 无 |
| Phase 5 | PRD 汇总 | `agents/product/phases/05-summary.md` | 完整 PRD 文档 | 无 |

## 用户故事格式规范

```
### US-{编号}：{用户故事标题}

**作为** {角色}，
**我希望** {功能描述}，
**以便** {业务价值}。

**验收标准：**
1. {具体的可测试条件}
2. {具体的可测试条件}
3. {具体的可测试条件}

**优先级：** P0 / P1 / P2
**预估工作量：** {S / M / L / XL}
```

## 功能优先级矩阵

| 维度 | P0（必须有） | P1（应该有） | P2（可以有） |
|------|------------|------------|------------|
| 定义 | MVP 必备，缺少则产品不可用 | 提升核心体验，显著增加价值 | 锦上添花，可延后 |
| 发布节奏 | MVP 版本 | 第 2-3 次迭代 | 后续迭代 |
| 验收标准 | 必须有自动化测试覆盖 | 建议有自动化测试 | 手动测试即可 |

## PRD 质量检查清单

- [ ] 每个用户故事都有明确的验收标准
- [ ] 验收标准可转化为测试用例（QA Agent 可消费）
- [ ] 非功能性需求已定义（性能、安全、可用性）
- [ ] MVP 范围明确且可在预期时间内交付
- [ ] 已标注与 Design / Spec / QA 的衔接点
- [ ] 风险清单已识别且有应对策略
