# 编码引导通用工作流

本文件定义所有 `*-coding-guide` 类型 skill 的共享工作流。各域 SKILL.md 通过 `workflow` 字段引用本模板，并声明域参数。

## 何时使用
1. AI 在对应域项目中编写或修改业务代码时（自动触发或手动指定）。
2. 人类开发者需要了解当前编码场景应遵循哪些规范时。
3. 由 `$task-router` 调度时。

## 执行原则
1. 以域 SKILL.md 中声明的 `scenario_map` 为场景路由表。
2. 始终加载域 SKILL.md 中声明的 `baseline_files`，保证最低合规。
3. 按编码场景加载 2~4 个命中文件，总加载量不超过域 SKILL.md 中声明的 `max_load`。
4. 涉及跨域时触发域 SKILL.md 中声明的 `cross_domain` 对应 skill。
5. 以域 SKILL.md 中声明的 `rules_index` 为规则索引真源。

## 标准工作流（必须执行）

### 1. 识别编码场景
- 从当前任务、代码上下文、用户指令中提取编码动作。
- 如域 SKILL.md 声明了 `context`（如应用类型、框架、UI 框架、部署模式），先确定上下文。
- 若任务包含多个场景，逐个处理，每个场景独立加载规则。

### 2. 查表确定加载集
- 读取域 SKILL.md 声明的 `scenario_map`。
- 先扫描场景标题列表（## A ~ ## Q），仅展开匹配的场景章节，跳过不相关的场景详情。
- 匹配当前编码场景，获取：始终加载文件 + 场景主文件 + 关联文件。
- 如有 profile/应用类型维度，加载对应的结构文件。

### 3. 读取规则并编码
- 按加载集读取规则文件。
- 编写代码时严格遵循已加载规则的 MUST 条款。
- SHOULD 条款尽量遵循，无法满足时注释说明原因。

### 4. 跨域联动
- 检查域 SKILL.md 中声明的 `cross_domain` 触发条件，命中时触发对应 skill。

### 5. 输出规范附录
- 编码完成后附"本次遵循的规则清单"。
- 标注跨域联动建议（如有）。
- 按 `agents/protocols/execution-trace.md` 格式，在输出末尾附执行追溯。
