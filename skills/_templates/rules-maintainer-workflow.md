# 规则维护通用工作流

本文件定义所有 `*-rules-maintainer` 类型 skill 的共享工作流。各域 SKILL.md 通过 `workflow` 字段引用本模板，并声明域参数。

## 执行原则
1. 以域 SKILL.md 声明的 `rules_index` 为规则入口与索引真源。
2. 以"单一来源"维护规则：同一约束只在一个文件定义，其他文件仅引用。
3. 以最小改动实现目标：只改被需求命中的文件，避免无关重写。
4. 以可验证为交付标准：修改后必须完成脚本校验与人工复核要点。

## 最小加载顺序
1. 先读取域 SKILL.md 声明的 `rules_index`。
2. 再读取命中范围对应文件。
3. 按需读取域 SKILL.md 声明的参考资料（`scope_map`/`load_map`、`structure`、`change_modes`、`output_contract`、`checklist`）。
4. 如涉及跨域，按域 SKILL.md 声明的 `cross_domain_file` 加载跨域规则。

## 标准工作流（必须执行）
1. **识别意图与模式**：规则新增 / 重构迁移 / 一致性审计 / 冲突修复。
2. **建立规则落点映射**：使用域 SKILL.md 声明的 `scope_map`（或 `load_map`）将需求映射到唯一目标文件。若映射到多个文件，显式声明主定义文件与引用文件。
3. **执行最小补丁**：仅修改命中文件。如新增文件，必须同步更新 `index.md`。
4. **执行结构校验**：运行域 SKILL.md 声明的 `validate_script`。
5. **执行语义校验**：运行域 SKILL.md 声明的 `lint_script`。
6. **处理冲突与阻塞**：若与现有规则冲突，先报告冲突点、影响面、备选方案。适用优先级：域 SKILL.md 声明的 `priority`。
7. **输出交付结论**：使用域 SKILL.md 声明的 `output_contract` 格式输出变更清单、命中规则、校验结果、P0/P1 风险、后续建议。
8. **执行追溯**：按 `agents/protocols/execution-trace.md` 格式，在输出末尾附执行追溯。

## 编写规则要求
1. 优先使用短句、可检查表述（如"必须""禁止""应当"）。
2. `profile` 文件只写相对 `common` 的差异，不重复通用内容。
3. 使用明确路径、命名与边界说明。
4. 除非用户指定框架，否则保持实现中立。
5. 未经用户明确要求，不主动改变既有行为边界。
