# 项目脚手架通用工作流

本文件定义所有 `*-project-scaffold` 类型 skill 的共享工作流。各域 SKILL.md 通过 `workflow` 字段引用本模板，并声明域参数。

## 执行原则
1. 以域 SKILL.md 声明的 `input_type` 为入口，自动映射所需规则与模板。
2. 生成的结构必须符合对应 profile/结构文件定义。
3. 生成的配置必须符合 `common/*.md` 通用规则。
4. 输出可直接使用的项目骨架，而非文档说明。

## 标准工作流（必须执行）
1. **确认模式/类型**：根据域 SKILL.md 声明的 `input_type` 和 `supported_modes` 确认用户选择。用户输入或从上下文推断。
2. **加载规则**：按域 SKILL.md 声明的 `scaffold_map` 映射读取规则与模板文件。
3. **生成目录结构**：按对应 profile/结构文件创建标准目录。
4. **生成配置文件**：按域 SKILL.md 声明的 `config_rules`（如有）生成各类配置。
5. **生成基础代码**：生成入口文件、健康检查、错误处理等基础结构。
6. **输出初始化报告**：已创建文件清单 + 下一步手动操作指引。
7. **执行追溯**：按 `agents/protocols/execution-trace.md` 格式，在输出末尾附执行追溯。
