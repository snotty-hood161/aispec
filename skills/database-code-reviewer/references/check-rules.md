# 可自动检查的规则条目清单

本文件列出可在数据库变更审查中自动/半自动检查的规则条目。

## 使用方式
1. 根据变更文件路径与内容，匹配下方检查项。
2. 每项标注检查方式：`模式匹配`（文件名/路径/内容正则）/ `人工审查`。
3. P0 项必须全部通过，P1 项允许带条件通过。

---

## 一、Schema 初始化脚本（database/database.md §1）

| ID | 级别 | 检查项 | 检查方式 |
|----|------|--------|----------|
| DB-01 | P0 | schema.sql 包含所有表结构、索引、菜单、权限初始化数据 | 人工审查：检查 schema.sql 完整性 |
| DB-02 | P0 | schema.sql 可在生产环境直接执行完成初始化（无外部依赖、无变量替换） | 人工审查：验证脚本可独立执行 |

## 二、迁移脚本管理（database/database.md §2）

| ID | 级别 | 检查项 | 检查方式 |
|----|------|--------|----------|
| DB-03 | P0 | 所有结构或数据变更通过新增迁移脚本完成，禁止直接修改 schema.sql | 模式匹配：检查 diff 中 schema.sql 是否被修改 |
| DB-04 | P0 | 迁移脚本位于 `docs/migrations/` 目录 | 模式匹配：检查新增 SQL 文件路径 |
| DB-05 | P0 | 迁移脚本命名格式为 `yyyyMMdd_版本号_变更说明.sql` | 模式匹配：正则 `^\d{8}_\d+_.+\.sql$` |
| DB-06 | P1 | 迁移脚本中日期与版本号递增，不与已有脚本冲突 | 模式匹配：比较已有脚本文件名排序 |
| DB-07 | P1 | 迁移脚本包含变更说明注释（脚本头部或文件名体现） | 人工审查：检查脚本内容与命名清晰度 |

## 三、历史脚本保护（database/database.md §3）

| ID | 级别 | 检查项 | 检查方式 |
|----|------|--------|----------|
| DB-08 | P0 | 严禁修改任何已存在的 SQL 脚本（含 schema.sql 和历史 migration） | 模式匹配：检查 diff 中已有 SQL 文件是否被修改 |
| DB-09 | P0 | 数据库变更仅通过新增脚本完成，禁止删除或覆盖历史脚本 | 模式匹配：检查 diff 中已有 SQL 文件是否被删除 |

## 四、SQL 质量（通用最佳实践）

| ID | 级别 | 检查项 | 检查方式 |
|----|------|--------|----------|
| DB-10 | P1 | DDL 变更包含 IF NOT EXISTS / IF EXISTS 防护，避免重复执行失败 | 模式匹配：搜索 CREATE/ALTER/DROP 语句是否有防护 |
| DB-11 | P1 | 大表 DDL 变更附性能影响评估（锁表时间、在线 DDL 方案） | 人工审查：检查变更影响说明 |
| DB-12 | P1 | 索引命名规范化（如 `idx_表名_列名`），禁止匿名索引 | 模式匹配：检查 CREATE INDEX 语句命名 |

## 跨域检查触发条件

| 条件 | 追加审查 |
|------|----------|
| 变更涉及 API 接口字段映射 | 追加 `rules/frontend-backend-collaboration.md` 契约规则 |
| 变更涉及权限/菜单初始化数据 | 追加 `rules/frontend/applications/admin-console.md` 权限规则 |
| 变更涉及枚举/状态码定义 | 追加 `rules/frontend-backend-collaboration.md` 错误码协同规则 |
