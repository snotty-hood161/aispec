# Phase 3 — 视觉设计（UI Design）

## 阶段目标
建立完整的设计系统（Design Token + 组件库），完成所有页面的视觉设计并输出可实施的设计方案。

## 执行流程

### 3.1 设计 Token 定义
**动作**：使用 Pencil MCP 的 `set_variables` 工具定义设计 Token。

#### 色彩体系
```
色彩 Token 结构：
├── 品牌色
│   ├── primary-50 ~ primary-900（主色 10 级梯度）
│   ├── secondary-50 ~ secondary-900（辅助色梯度）
│   └── accent（强调色）
├── 语义色
│   ├── success（成功）
│   ├── warning（警告）
│   ├── error（错误）
│   └── info（信息）
├── 中性色
│   ├── gray-50 ~ gray-900（灰阶梯度）
│   ├── background-primary（主背景）
│   ├── background-secondary（次背景）
│   └── background-elevated（抬升背景）
└── 文本色
    ├── text-primary（主文本）
    ├── text-secondary（辅助文本）
    ├── text-placeholder（占位文本）
    └── text-disabled（禁用文本）
```

#### 字体排印
```
字体 Token 结构：
├── 字体族
│   ├── font-display（展示字体 — 标题、大号文字）
│   ├── font-body（正文字体 — 段落、表单）
│   └── font-mono（等宽字体 — 代码、数据）
├── 字号
│   ├── text-xs（12px）
│   ├── text-sm（14px）
│   ├── text-base（16px）
│   ├── text-lg（18px）
│   ├── text-xl（20px）
│   ├── text-2xl（24px）
│   ├── text-3xl（30px）
│   └── text-4xl（36px）
├── 行高
│   ├── leading-tight（1.25）
│   ├── leading-normal（1.5）
│   └── leading-relaxed（1.75）
└── 字重
    ├── font-regular（400）
    ├── font-medium（500）
    ├── font-semibold（600）
    └── font-bold（700）
```

#### 间距系统
```
间距 Token（4px 基准倍数）：
├── space-1（4px）
├── space-2（8px）
├── space-3（12px）
├── space-4（16px）
├── space-5（20px）
├── space-6（24px）
├── space-8（32px）
├── space-10（40px）
├── space-12（48px）
└── space-16（64px）
```

#### 圆角与阴影
```
圆角 Token：
├── radius-none（0px）
├── radius-sm（4px）
├── radius-md（8px）
├── radius-lg（12px）
├── radius-xl（16px）
└── radius-full（9999px）

阴影 Token：
├── shadow-sm（轻微抬升）
├── shadow-md（中等抬升）
├── shadow-lg（明显抬升）
└── shadow-xl（浮层/弹窗）
```

### 3.2 基础组件设计
**动作**：使用 Pencil MCP 工具设计基础组件。

必须设计的组件清单：

| 组件类别 | 组件 | 状态变体 |
|---------|------|---------|
| 按钮 | 主按钮、次按钮、文字按钮、图标按钮 | 默认/悬停/按下/禁用/加载中 |
| 输入 | 文本框、文本域、下拉选择、日期选择、搜索框 | 默认/聚焦/错误/禁用 |
| 数据展示 | 表格、卡片、列表项、标签、徽章 | 不同密度/不同状态 |
| 反馈 | Toast、对话框、抽屉、通知 | 成功/警告/错误/信息 |
| 导航 | 顶部导航、侧边菜单、标签页、面包屑 | 选中/未选中/折叠 |
| 布局 | 页面容器、分栏布局、分割线 | 不同宽度/间距 |

每个组件设计要求：
- 多状态展示（默认、悬停、激活、禁用）。
- 标注尺寸（宽高、内边距、间距）。
- 标注使用的 Token（颜色变量、字体变量）。
- 设为可复用组件（`reusable: true`）。

### 3.3 页面设计
**动作**：逐页设计所有页面。

设计顺序：
1. **布局模板** — 先设计 2-3 种布局模板（带侧边栏、不带侧边栏、全宽等）。
2. **关键页面** — 首页/仪表盘、核心功能页面（按 PRD 优先级）。
3. **通用页面** — 登录/注册、404/500、空状态。
4. **辅助页面** — 设置、帮助等。

每个页面设计要求：
- 基于 Phase 2 的信息架构和页面清单。
- 使用已定义的设计 Token 和组件。
- 标注页面标题和路径。
- 标注关键交互点。

### 3.4 主题支持（如需要）
**动作**：设计深色模式或多主题方案。

- 使用 `set_variables` 定义主题变量。
- 确保语义色在不同主题下的对比度满足 WCAG AA（4.5:1）。
- 为每个核心页面验证深色模式效果。

### 3.5 动效与微交互规范
**动作**：定义关键动效规范。

```
动效 Token：
├── duration
│   ├── duration-fast（150ms）— 小元素状态切换
│   ├── duration-normal（300ms）— 常规过渡
│   └── duration-slow（500ms）— 页面级过渡
├── easing
│   ├── ease-in-out — 常规过渡
│   ├── ease-out — 元素进入
│   └── ease-in — 元素退出
└── 动效场景
    ├── 页面切换 — 淡入淡出 / 滑动
    ├── 弹窗显示 — 缩放+淡入
    ├── 列表加载 — 逐项入场（交错延迟）
    ├── 按钮点击 — 微缩反馈
    └── 悬停效果 — 抬升 + 阴影变化
```

## 阶段转换条件
以下条件满足后进入 Phase 4：
1. 设计 Token 已完整定义。
2. 基础组件已设计完成（至少覆盖按钮、输入、表格、卡片）。
3. 核心页面已设计完成（至少 MVP 的核心页面）。
4. 通用页面已设计（登录/注册/空状态/错误页）。
