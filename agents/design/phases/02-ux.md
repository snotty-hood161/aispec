# Phase 2 — 交互设计（UX Design）

## 阶段目标
定义产品的信息架构、用户操作流程和交互逻辑，为视觉设计提供结构基础。

## 引导问题清单

### 2.1 信息架构设计
**动作**：定义产品的页面层级和导航结构。

设计要素：
1. **一级导航** — 产品的顶层功能分区（如首页、工作台、消息、设置）。
2. **二级导航** — 每个分区下的子页面/子功能。
3. **页面层级** — 从入口到最深层的层级数量（建议不超过 4 级）。
4. **导航模式** — 顶部导航 / 侧边导航 / 底部标签栏 / 混合模式。

输出格式（文本化信息架构）：
```
首页
├── 仪表盘
│   ├── 数据概览
│   └── 快捷操作
├── {模块A}
│   ├── 列表页
│   ├── 详情页
│   └── 编辑页
├── {模块B}
│   └── ...
└── 设置
    ├── 个人信息
    ├── 安全设置
    └── 系统配置
```

### 2.2 用户流程设计
**动作**：为每个核心场景设计操作流程。

每个流程包含：
1. **触发点** — 用户从哪里开始。
2. **操作步骤** — 逐步的操作动作。
3. **决策点** — 用户需要做选择的节点。
4. **结束状态** — 操作完成后的反馈。
5. **异常路径** — 出错时的处理流程。

流程描述格式（使用 Mermaid 或文本）：
```
核心场景：{场景名}
1. 用户进入 {页面}
2. 点击 {操作} → 显示 {结果/弹窗/新页面}
3. 填写 {信息} → 校验 {规则}
   - 校验通过 → 4
   - 校验失败 → 显示错误提示，返回 3
4. 确认提交 → 显示成功反馈
5. 自动跳转到 {目标页面}
```

### 2.3 页面清单
**动作**：列出产品的完整页面清单。

页面分类：
- **通用页面**：登录、注册、忘记密码、404、500、无权限。
- **核心页面**：各功能模块的列表页、详情页、编辑页、创建页。
- **辅助页面**：设置、帮助、关于、引导。

每个页面标注：
- 页面名称与路径
- 页面类型（列表/详情/表单/仪表盘/空页面）
- 页面间的跳转关系
- 需要的核心组件

### 2.4 交互规则定义
**动作**：定义全局性的交互规则。

规则类型：
1. **加载策略** — 骨架屏 / 加载动画 / 渐进加载。
2. **空状态** — 无数据时的引导设计。
3. **错误反馈** — 表单校验、操作失败、网络异常的反馈方式。
4. **成功反馈** — 操作成功的确认方式（Toast / 页面跳转 / 内联提示）。
5. **确认机制** — 危险操作（删除、批量操作）的二次确认。
6. **搜索与筛选** — 即时搜索 vs 提交搜索、筛选器样式。
7. **分页与加载更多** — 翻页 vs 无限滚动 vs 加载更多。
8. **多选与批量操作** — 选择模式与批量操作栏。

### 2.5 响应式策略
**问题**：产品需要适配哪些端？

策略定义：
- **断点定义**：手机（<768px）、平板（768-1024px）、桌面（>1024px）、大屏（>1440px）。
- **布局变化**：导航折叠/展开、列数变化、元素隐藏/显示。
- **交互适配**：触控 vs 鼠标、长按 vs 右键、滑动 vs 滚动。

## 阶段转换条件
以下条件满足后进入 Phase 3：
1. 信息架构已定义，导航模式已确认。
2. 核心场景（至少 3 个）的操作流程已设计。
3. 页面清单已完成。
4. 全局交互规则已定义。
