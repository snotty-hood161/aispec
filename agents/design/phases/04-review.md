# Phase 4 — 设计验证（Design Review）

## 阶段目标
验证设计的完整性、一致性和可实现性，输出设计交付文档，确保开发团队可以准确还原。

## 验证清单

### 4.1 设计完整性检查
- [ ] 所有 PRD 中的 P0 功能对应的页面已设计。
- [ ] 通用页面（登录/注册/404/500/空状态）已覆盖。
- [ ] 所有核心用户流程的每个步骤都有对应页面。
- [ ] 响应式适配方案已为关键断点设计（如需要）。

### 4.2 设计一致性检查
**动作**：使用 `get_screenshot` 和 `snapshot_layout` 工具验证。

- [ ] 所有页面使用的颜色均来自已定义的色彩 Token。
- [ ] 字体使用严格遵循字号层级（不随意使用未定义的字号）。
- [ ] 间距使用 4px 倍数基准。
- [ ] 圆角使用已定义的 Token 值。
- [ ] 同类元素在不同页面中外观一致（如所有主按钮样式相同）。
- [ ] 图标风格统一（线性/填充/双色不混用）。

### 4.3 可用性审查
- [ ] 关键操作路径在 3 步以内完成。
- [ ] 操作按钮位置符合直觉（如确认在右、取消在左）。
- [ ] 表单标签清晰，错误提示明确。
- [ ] 空状态有引导操作。
- [ ] 加载状态有明确反馈。
- [ ] 危险操作有二次确认。

### 4.4 无障碍检查
- [ ] 文本与背景的对比度 ≥ 4.5:1（WCAG AA）。
- [ ] 大号文本（≥18px 粗体或 ≥24px 常规）对比度 ≥ 3:1。
- [ ] 可交互元素的触控区域 ≥ 44x44px。
- [ ] 颜色不是传达信息的唯一手段（辅以图标/文字）。
- [ ] 焦点状态可见（键盘导航）。

### 4.5 设计还原度审查（开发后使用）
**动作**：对比设计稿与实际实现的页面。

审查维度：
- 布局结构是否一致（元素位置、间距）。
- 色彩是否准确（色值偏差 ≤ 2%）。
- 字体是否正确（字体族、字号、行高、字重）。
- 组件样式是否一致（圆角、阴影、边框）。
- 动效是否实现（过渡时长、缓动曲线）。
- 响应式断点是否正确（各断点下布局变化）。

还原度评分标准：
| 评分 | 还原度 | 说明 |
|------|--------|------|
| A | ≥ 95% | 优秀，细节精准 |
| B | 85-94% | 良好，少量微调 |
| C | 70-84% | 合格，需要修正 |
| D | < 70% | 不合格，需要重做 |

## 阶段完成条件
1. 设计审查报告中无 P0 级问题。
2. 设计交付文档已生成。
3. CSS Token 已导出。
4. 用户确认设计方案。
