# Phase 1 — 设计调研（Design Research）

## 阶段目标
理解产品需求上下文、分析竞品设计模式、确定设计风格方向和视觉调性。

## 输入依赖
- Product Agent 的 PRD 文档（用户画像、用户故事、功能清单）。
- 如无 PRD，Design Agent 需先引导用户明确核心需求。

## 引导问题清单

### 1.1 产品理解
**动作**：深入理解 PRD 中与设计相关的信息。

聚焦维度：
- 产品类型（SaaS/工具/社交/电商/内容/管理后台...）。
- 目标用户特征（年龄、技术水平、使用设备、使用频率）。
- 核心用户场景（用户最频繁的操作是什么）。
- 产品调性（专业/轻松/严肃/活泼/极简/丰富）。

### 1.2 竞品设计分析
**动作**：使用浏览器工具实地分析竞品界面设计。

每个竞品的设计分析维度：
1. **整体风格** — 色彩体系、明暗倾向、视觉密度。
2. **布局模式** — 导航位置（顶部/侧边/底部）、内容区域划分、栅格系统。
3. **字体策略** — 字体族、字号层级、行高、字重变化。
4. **组件设计** — 按钮风格、卡片样式、表单设计、表格样式。
5. **交互模式** — 动画/过渡效果、加载策略、空状态处理。
6. **亮点与不足** — 设计上值得借鉴和需要改进的地方。

截图记录：
- 首页/着陆页
- 核心功能页面（2-3 个）
- 移动端适配（如有）
- 特色交互效果

### 1.3 设计风格定位
**问题**：这个产品的视觉调性应该是什么？

引导要点（用户需从以下方向选择或组合）：
- **极简克制** — 大量留白、精确排版、少即是多，靠细节取胜。
- **丰富表现** — 色彩鲜明、动效丰富、视觉冲击力强。
- **专业严谨** — 信息密度高、数据可视化、功能导向。
- **温暖亲和** — 圆润造型、柔和配色、引导性强。
- **高端精致** — 克制用色、精良排版、质感细节。
- **活力潮流** — 大胆配色、创意布局、年轻化表达。
- **复古怀旧** — 经典元素、怀旧配色、温情质感。
- **未来科技** — 深色主题、霓虹色彩、科技感元素。

### 1.4 品牌色与基调确认
**问题**：有没有已确定的品牌色或视觉规范？

引导要点：
- 是否有已有的品牌色/Logo/VI 手册。
- 如果没有，根据产品调性推荐配色方案。
- 确认明暗偏好（浅色模式 / 深色模式 / 双主题支持）。
- 确认圆角偏好（方正 vs 圆润）。

## 阶段转换条件
以下条件满足后进入 Phase 2：
1. 产品需求上下文已理解。
2. 至少 2 个竞品完成设计分析。
3. 设计风格方向已确认。
4. 主色调已初步确定。
