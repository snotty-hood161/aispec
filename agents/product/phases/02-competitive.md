# Phase 2 — 竞品深度分析（Competitive Analysis）

## 阶段目标
对 Phase 1 识别的竞品进行深度分析，提取功能清单、优劣势、用户体验洞察，明确差异化定位。

## 引导问题清单

### 2.1 竞品功能梳理
**动作**：对每个核心竞品逐一深入分析。

每个竞品的分析维度：
1. **核心功能清单** — 列出产品的主要功能模块。
2. **功能亮点** — 哪些功能做得特别好？用户评价最高的是什么？
3. **体验细节** — 使用浏览器工具实际体验产品，截图记录：
   - 首页/落地页设计
   - 核心操作流程（注册→首次使用→核心功能）
   - 关键页面的交互设计
   - 移动端适配（如有）
4. **技术特征** — 加载速度、响应性、离线支持、API 开放性等。

### 2.2 优劣势矩阵
**问题**：每个竞品的核心优势和明显短板是什么？

分析框架：
- **优势**：功能完整性、用户体验、品牌认知、生态整合、价格优势
- **劣势**：功能缺失、体验痛点、价格过高、本地化不足、扩展性差
- **威胁**：竞品的发展势头、资本支持、技术壁垒
- **机会**：竞品忽略的用户群体、未满足的需求、技术代差

### 2.3 差异化定位
**问题**：我们的产品与竞品相比，核心差异化在哪里？

引导要点：
- 不要在所有维度上与竞品竞争，选 1-3 个核心差异点。
- 差异化可以来自：功能创新、体验升级、价格策略、目标用户细分、技术优势、本地化优势。
- 用一句话定义差异化定位（如"面向中小团队的轻量级 XX 工具"）。

### 2.4 功能对标矩阵
**动作**：构建功能对比表，标注"有/无/优/弱"。

```
| 功能点 | 竞品A | 竞品B | 竞品C | 我们的计划 |
|--------|-------|-------|-------|-----------|
| 功能1  | ★★★   | ★★    | ★     | ★★★★（核心差异） |
| 功能2  | ★★    | ★★★   | ★★    | ★★（跟随） |
| 功能3  | ✗     | ★     | ✗     | ★★★（蓝海机会） |
```

★ = 基础支持  ★★ = 良好  ★★★ = 优秀  ★★★★ = 行业领先  ✗ = 不支持

## 阶段转换条件
以下条件满足后进入 Phase 3：
1. 至少 3 个竞品完成深度分析。
2. 功能对标矩阵已构建。
3. 差异化定位已明确。
