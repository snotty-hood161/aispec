# Phase 1 — 市场调研（Research）

## 阶段目标
了解市场现状、识别竞品、梳理行业趋势，为产品定位提供信息基础。

## 引导问题清单

### 1.1 产品方向确认
**问题**：你想做的产品/系统是什么方向？

引导要点：
- 用一句话描述产品核心价值。
- 这个产品解决什么问题？为谁解决？
- 你为什么想做这个产品？（个人需求、商业机会、技术探索等）

### 1.2 已知竞品收集
**问题**：你知道市面上有哪些类似的产品或系统吗？

引导要点：
- 用户可能已经有初步了解的竞品，先收集。
- 如果用户不清楚，Product Agent 主动使用搜索工具调研。
- 区分直接竞品（同类产品）和间接竞品（替代方案）。

### 1.3 主动市场调研
**动作**：Product Agent 使用工具主动调研。

调研步骤：
1. 使用 WebSearch 搜索产品方向的关键词，了解市场格局。
2. 识别 3-5 个核心竞品（国内外各选代表）。
3. 搜索行业报告、用户评价、媒体分析。
4. 使用浏览器工具访问竞品官网，获取第一手信息。

调研维度：
- 产品定位与目标用户
- 核心功能与卖点
- 商业模式（免费/付费/SaaS/开源）
- 技术架构（如有公开信息）
- 用户评价与口碑（应用商店评分、社交媒体反馈）
- 融资与团队背景（如有公开信息）

### 1.4 行业趋势
**问题**：这个领域近期有什么技术或市场趋势？

引导要点：
- AI/大模型对这个领域的影响。
- 用户需求的变化趋势。
- 监管或合规方面的新要求。
- 新技术带来的可能性（如 AI、AR/VR、物联网等）。

## 阶段输出

完成本阶段后，Product Agent 输出以下内容：

```
## 市场调研报告

### 产品方向
{一段话描述产品方向与核心价值}

### 竞品清单
| 竞品名称 | 类型 | 定位 | 目标用户 | 商业模式 | 备注 |
|---------|------|------|---------|---------|------|
| {竞品1} | 直接竞品 | {定位} | {用户} | {模式} | {备注} |
| {竞品2} | 间接竞品 | {定位} | {用户} | {模式} | {备注} |

### 行业趋势摘要
- {趋势1}
- {趋势2}
- {趋势3}

### 关键洞察
- {洞察1：市场机会}
- {洞察2：用户痛点}
- {洞察3：技术趋势}
```

## 阶段转换条件
以下条件满足后进入 Phase 2：
1. 至少识别出 3 个竞品。
2. 产品方向已明确。
3. 行业趋势有初步了解。
