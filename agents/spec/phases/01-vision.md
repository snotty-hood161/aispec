# Phase 1 — 项目愿景（Vision）

## 阶段目标
明确系统的核心定位、目标用户和功能范围，为后续技术决策提供业务基础。

## 引导问题清单

### 1.1 核心问题定义
**问题**：这个系统/项目要解决什么核心问题？

引导要点：
- 用一句话描述系统的核心价值主张。
- 当前这个问题是怎么被解决的？现有方案的痛点是什么？
- 如果不做这个系统，最大的损失是什么？

### 1.2 目标用户画像
**问题**：目标用户是谁？有哪些核心使用场景？

引导要点：
- 主要用户角色有哪些？（如管理员、普通用户、运营人员）
- 每个角色的核心诉求是什么？
- 用户的技术水平如何？（影响 UI/UX 复杂度决策）
- 预估用户规模（影响架构和性能决策）。
- 用户使用的设备/平台（影响客户端技术选型）。

### 1.3 核心功能清单
**问题**：核心功能有哪些？请按优先级排序。

引导要点：
- 列举所有功能点，按 P0/P1/P2 分级：
  - **P0 — 必须有**：没有这些功能系统无法上线。
  - **P1 — 应该有**：上线后第一个迭代需要补齐。
  - **P2 — 可以有**：未来规划，当前不阻塞。
- 每个功能点用一句话描述做什么。
- 功能之间有没有依赖关系？

### 1.4 竞品与差异化
**问题**：有没有竞品或参考产品？差异化在哪里？

引导要点：
- 市面上有哪些类似系统？
- 我们的核心差异化在哪里？（功能/体验/价格/生态）
- 有没有需要对标的技术指标？（如响应速度、并发量）

### 1.5 MVP 范围
**问题**：MVP（最小可用产品）需要包含哪些功能？

引导要点：
- 从 P0 功能中进一步精简，哪些是 MVP 必须的？
- MVP 的目标上线时间是什么？
- MVP 之后的迭代节奏是怎样的？

### 1.6 业务约束
**问题**：有没有不可协商的业务约束？

引导要点：
- 团队规模和技能分布（影响技术选型的可行性）。
- 开发周期和里程碑要求。
- 预算范围（影响基础设施和第三方服务选择）。
- 已有系统的集成需求。

## 阶段输出

完成本阶段后，Spec Agent 输出以下内容：

```

### 系统目标
{一段话描述系统核心价值}

### 用户画像
| 角色 | 核心诉求 | 使用设备 | 技术水平 |
|------|---------|---------|---------|
| {角色名} | {诉求} | {设备} | {水平} |

### 核心功能矩阵
| 功能 | 优先级 | 描述 | 依赖 |
|------|--------|------|------|
| {功能名} | P0/P1/P2 | {一句话描述} | {依赖的功能} |

### MVP 范围
- {MVP 包含的功能列表}
- 目标上线时间：{时间}

### 业务约束
- 团队：{规模与技能}
- 周期：{时间约束}
- 预算：{预算范围}

### 竞品参考
| 竞品 | 优势 | 劣势 | 我们的差异化 |
|------|------|------|------------|
| {竞品名} | {优势} | {劣势} | {差异化} |
```

## 阶段转换条件
以下条件满足后进入 Phase 2：
1. 系统目标已明确定义。
2. 至少识别出一个用户角色和核心场景。
3. P0 功能清单已确定。
4. MVP 范围已划定。
