# Phase 3 — 全局约束（Constraints）

## 阶段目标
定义系统的横切关注点和全局性约束条件。这些约束将作为后续模块设计和技术实现的刚性边界。

## 约束维度清单

### 3.1 安全约束
**问题**：系统在安全方面需要满足哪些要求？

引导要点：
- 认证安全：密码策略、多因素认证、会话管理。
- 授权安全：最小权限原则、权限校验粒度。
- 数据安全：传输加密（TLS）、存储加密（AES）、敏感字段脱敏。
- 输入安全：SQL 注入/XSS/CSRF 防护策略。
- 审计安全：操作日志记录范围、审计追溯要求。
- API 安全：限流、签名、IP 白名单。

输出格式：
```
### 安全约束
| 维度 | 要求 | 级别 | 实现方案 |
|------|------|------|---------|
| 传输加密 | 全站 HTTPS | 必须 | TLS 1.3 |
| ... | ... | ... | ... |
```

### 3.2 性能约束
**问题**：系统需要达到什么性能指标？

引导要点：
- 接口响应时间：P50/P95/P99 目标值。
- QPS（每秒查询数）：峰值和日常负载。
- 吞吐量：批量操作/报表生成的预期耗时。
- 并发用户数：同时在线用户数。
- 数据量级：预估存储量增长曲线。
- 冷启动时间：Serverless 场景需关注。

输出格式：
```
### 性能约束
| 指标 | 目标值 | 当前基线 | 优化策略 |
|------|--------|---------|---------|
| API P95 响应时间 | < 200ms | — | 缓存 + 索引优化 |
| 峰值 QPS | 5000 | — | 水平扩展 + 限流 |
| ... | ... | ... | ... |
```

### 3.3 可用性约束
**问题**：系统的可用性要求是什么？

引导要点：
- SLA 目标（99.9% / 99.95% / 99.99%）。
- 允许的最长不可用时间（月/年）。
- 容灾策略：单机房 / 同城双活 / 异地灾备。
- 降级策略：核心功能降级方案、非核心功能熔断方案。
- 故障恢复时间目标（RTO）和数据恢复点目标（RPO）。

### 3.4 可扩展性约束
**问题**：系统未来需要如何扩展？

引导要点：
- 水平扩展能力：能否通过加机器解决性能瓶颈？
- 垂直扩展预期：单机资源上限。
- 数据分片策略：是否需要分库分表？分片键选择。
- 多租户扩展：租户隔离策略（共享数据库 / 独立数据库 / Schema 隔离）。
- 功能扩展：插件化/模块化扩展机制。

### 3.5 合规约束
**问题**：系统需要满足哪些合规要求？

引导要点：
- 数据隐私法规（GDPR / CCPA / 个人信息保护法）。
- 行业合规标准（PCI-DSS / HIPAA / 等保）。
- 数据存储地域要求（数据不出境 / 特定区域存储）。
- 用户数据权利（查看/导出/删除/可携带性）。
- 内容合规要求（UGC 审核 / 敏感词过滤）。

### 3.6 成本约束
**问题**：基础设施和运维的预算范围是什么？

引导要点：
- 月度/年度基础设施预算。
- 第三方服务费用预算（SaaS 工具、API 调用费等）。
- 是否有成本优化的优先级排序（如用开源替代商业方案）。
- 许可证成本考虑（开源 / 商业许可 / 按量付费）。

### 3.7 部署约束
**问题**：系统如何部署和交付？

引导要点：
- 部署环境：公有云 / 私有云 / 混合云 / 客户私有化。
- 容器化策略：Docker / Kubernetes / 裸部署。
- CI/CD 流水线：自动化构建、测试、部署要求。
- 环境规划：开发 / 测试 / 预发 / 生产环境拓扑。
- 发布策略：蓝绿部署 / 滚动更新 / 灰度发布。
- 回滚策略：回滚触发条件和回滚流程。

### 3.8 第三方集成约束
**问题**：系统需要对接哪些外部系统或服务？

引导要点：
- 需要集成的外部系统清单。
- 每个外部系统的 SLA 和可用性。
- 外部系统不可用时的降级方案。
- 数据同步策略（实时 / 准实时 / 离线批量）。
- 接口协议和认证方式。

### 3.9 数据策略约束
**问题**：数据的全生命周期如何管理？

引导要点：
- 数据分类分级（公开/内部/机密/绝密）。
- 备份策略：全量/增量频率、保留周期。
- 归档策略：冷热数据分离条件。
- 数据生命周期：过期数据处理策略。
- 数据一致性要求：强一致/最终一致的适用场景。

### 3.10 监控与可观测性约束
**问题**：如何监控系统的运行状态和健康度？

引导要点：
- 日志：日志级别、格式、集中采集方案。
- 指标：核心业务指标和技术指标。
- 告警：告警规则、通知渠道、值班策略。
- 链路追踪：分布式追踪需求和采样策略。
- 仪表盘：需要哪些核心仪表盘。

## 阶段转换条件
以下条件满足后进入 Phase 4：
1. 安全约束已定义（至少覆盖认证、授权、传输安全）。
2. 性能约束已定义（至少有 QPS 和响应时间目标）。
3. 部署约束已定义（至少确定部署环境）。
4. 其他约束维度可标注"按需补充"，但以上 3 项不可跳过。
