# 执行追溯摘要格式（MUST）

所有任务执行完毕后，无论单体模式还是多 Agent 模式，必须在输出末尾附上执行追溯摘要。目的是让用户清晰了解本次执行的完整调用链路。

## 单体模式格式

单个 AI 实例通过 skill 直接执行任务时，在任务输出末尾附：

```
## 执行追溯

| 项目 | 内容 |
|------|------|
| 模式 | 单体模式 |
| 调用 Skill | {skill 名称，如 $go-server-coding-guide} |
| 任务类型 | {coding / review / scaffold / rule-maintenance} |
| 加载规则 | {本次实际加载的规则文件列表，如 baseline.md, api-design.md, error-handling.md} |
| 跨域规则 | {如涉及跨域，列出跨域规则文件；无则写"无"} |
| 跨域联动 | {如需其他域配合，说明建议；无则写"无"} |
```

## 多 Agent 模式格式

Coordinator 编排多个域 Agent 执行任务时，在汇总输出末尾附：

```
## 执行追溯

| 项目 | 内容 |
|------|------|
| 模式 | 多 Agent 模式 |
| 执行顺序 | {实际执行顺序，标注串行/并行，如 Database(串行) → GoServer(串行) → Collaboration(串行) → Frontend(串行)} |

### 各域追溯

| Agent | 调用 Skill | 任务类型 | 加载规则 |
|-------|-----------|---------|---------|
| {agent 名称} | {skill 名称} | {任务类型} | {加载的规则文件列表} |
| ... | ... | ... | ... |

### 跨域交接

- {发起方 Agent → 接收方 Agent：交接内容摘要}
- {无交接则写"无"}
```

## 填写规则
1. **加载规则**：列出本次实际读取并遵循的规则文件名（不含 `rules/` 前缀），按加载顺序排列。
2. **跨域规则**：仅在涉及跨域协作时填写（如 `frontend-backend-collaboration.md`、`security/security-baseline.md`）。
3. **跨域联动**：说明需要其他域配合的事项，无则写"无"。
4. **多 Agent 模式的各域追溯表**：每个被调度的域 Agent 独立一行。
5. **跨域交接**：列出本次执行中所有实际发生的交接（按 `protocols/handoff.md` 格式），无则写"无"。
