# rules/go-server/common/forbidden.md

## 禁止事项
1. 禁止在 handler 中直接操作数据库。
2. 禁止忽略错误返回值（除显式注释说明的可忽略场景）。
3. 禁止新增无上限缓存、无上限队列或无超时外部调用。
4. 禁止未评审的破坏性接口变更和数据库结构变更。
5. 禁止将业务实体放入通用共享目录后被跨服务直接复用。
6. 禁止将系统错误（数据库驱动错误、RPC 原始错误、panic 堆栈）原样返回给调用方。
7. 禁止在多个 handler 中分散实现错误到响应的映射逻辑，必须走统一错误处理中间件。
8. 禁止在 `pkg` 中定义带业务语义的错误（如 `user_error`、`order_error`）。
9. 禁止硬编码数据库、Redis、MinIO 等外部依赖地址和凭据。
10. 禁止在 CORS 中间件中硬编码域名白名单。
11. 禁止将统计分析临时读模型用于常规 CRUD 查询与写入。
12. 禁止在 `init()` 中初始化数据库、Redis、MinIO、JWT 等外部组件。
13. 禁止通过包级全局可变单例暴露组件实例（如全局 `DB`、全局 `RedisClient`）。
14. 禁止在 handler/service 内直接构造基础组件客户端，必须通过 DI 注入。
15. 禁止对失败请求统一返回 `200` 并仅依赖响应体业务 `code` 区分错误。
