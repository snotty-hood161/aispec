# rules/go-server/profiles/microservice/gateway.md

## 文档目标
1. 定义 API 网关的职责边界与约束。仅在架构包含网关层时启用本文件。

---

## 网关职责（MUST）

1. 网关必须且仅承担以下职责：
   - 路由转发（请求路由到目标服务）
   - 协议转换（HTTP ↔ gRPC，若需要）
   - 鉴权（统一认证，Token 校验）
   - 限流（全局/接口级限流）
   - 日志与追踪（注入 trace ID，记录访问日志）
2. 网关禁止包含业务逻辑（数据聚合、字段转换、业务校验）。
3. 网关路由配置必须版本化，与服务契约保持同步。

检查方式：架构评审
阻断级别：阻断合并

---

## 网关选型与部署（MUST）

1. 推荐方案：`Kong`、`APISIX`、`Envoy`、`Traefik`，或自建轻量网关。
2. gRPC ↔ HTTP 转换推荐使用 `grpc-gateway`，从 Proto 自动生成，禁止手写转换逻辑。
3. 网关必须高可用部署（至少 2 实例），并配置健康检查。

### SHOULD
1. 网关配置通过配置中心管理，支持动态更新路由规则。
2. 网关指标（QPS、延迟、错误率）纳入统一监控。

检查方式：架构评审
阻断级别：阻断合并
