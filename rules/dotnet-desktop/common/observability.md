# rules/dotnet-desktop/common/observability.md

## 文档目标
1. 定义 C#/.NET 桌面应用日志、崩溃报告、使用遥测的统一约束。
2. 桌面应用与服务端不同，重点在于客户端日志留存、崩溃收集和用户行为分析。

---

## 日志（MUST）

1. 必须使用结构化日志（推荐 Serilog），禁止只打拼接字符串或使用 `Console.WriteLine`。
2. 日志必须写入本地文件，文件路径位于用户数据目录（`LocalApplicationData`），禁止写入程序安装目录。
3. 日志级别至少区分 `Debug`、`Information`、`Warning`、`Error`、`Fatal`。
4. 禁止输出敏感信息：密码、令牌、证件号、银行卡完整信息。
5. 日志必须包含时间戳（ISO 8601）、日志级别、来源类名。
6. 生产发布版默认日志级别为 `Information`，`Debug` 级别仅在排查问题时临时开启。
7. 日志文件必须配置滚动策略（按大小或日期），防止日志文件无限增长占满磁盘。
8. 使用 `ILogger<T>` 注入，禁止直接使用静态日志方法。

### 日志配置示例（Serilog）
```csharp
Log.Logger = new LoggerConfiguration()
    .MinimumLevel.Information()
    .MinimumLevel.Override("Microsoft", LogEventLevel.Warning)
    .WriteTo.File(
        path: Path.Combine(
            Environment.GetFolderPath(Environment.SpecialFolder.LocalApplicationData),
            "MyApp", "logs", "app-.log"),
        rollingInterval: RollingInterval.Day,
        retainedFileCountLimit: 30,
        fileSizeLimitBytes: 10 * 1024 * 1024)
    .CreateLogger();
```

### SHOULD
1. 关键用户操作日志（登录、数据修改、导入导出）单独标记，便于问题排查。
2. 提供日志导出功能，方便用户在反馈问题时附带日志文件。
3. 日志量过大时支持采样或级别过滤。

检查方式：代码审查
阻断级别：阻断合并

## 崩溃报告（MUST）

1. 应用崩溃时必须收集异常信息并持久化到本地崩溃报告文件。
2. 崩溃报告必须包含：
   - 异常类型、消息、完整堆栈跟踪。
   - 应用版本号、.NET 运行时版本、操作系统版本。
   - 崩溃时间、用户最近操作上下文（脱敏）。
3. 崩溃报告文件存储在用户数据目录，下次启动时检测到未上报的崩溃报告应提示用户上报。
4. 崩溃报告上报必须经用户同意（或在隐私协议中明确告知），禁止静默收集。

### SHOULD
1. 集成第三方崩溃收集服务（Sentry / AppCenter / 自建服务），实现自动聚合和告警。
2. 崩溃报告包含设备信息（CPU、内存、显卡、分辨率），便于定位环境相关问题。
3. 对频繁崩溃的版本自动触发告警通知开发团队。

检查方式：集成测试 + 崩溃模拟验证
阻断级别：阻断合并

---

## 使用遥测（SHOULD）

1. 遥测数据收集必须经用户同意，应用首次启动时明确告知并提供关闭选项。
2. 推荐收集的遥测数据：
   - 功能使用频率（哪些功能常用/不常用）。
   - 启动时间、页面加载时间等性能指标。
   - 操作系统版本和屏幕分辨率分布。
3. 遥测数据禁止包含用户个人信息或业务数据内容。
4. 遥测数据发送失败不应影响应用正常使用，必须异步且有重试上限。

检查方式：隐私合规审查
阻断级别：告警记录

---

## 诊断模式（SHOULD）

1. 应用提供诊断模式入口（如设置页隐藏按钮或命令行参数），启用后输出详细调试日志。
2. 诊断模式下可展示：当前内存使用、线程数、网络连接状态、缓存命中率等运行时信息。
3. 诊断模式仅在排查问题时临时启用，退出后自动恢复正常日志级别。
4. 提供"导出诊断包"功能，一键收集日志文件、崩溃报告、系统信息，供技术支持分析。

检查方式：功能测试
阻断级别：告警记录
