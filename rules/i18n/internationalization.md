# 国际化（i18n）规范

## Skill 协作
1. 各域 coding-guide 编写涉及国际化的代码时，自动加载本规则。
2. 本规则为跨域国际化基线，各域可在自身规则中补充平台特定细节。
3. 跨域业务任务使用 `$task-router` 自动路由。

## 适用范围
1. 计划支持多语言的产品必须从项目初始化阶段开始遵循本规范。
2. 仅支持单语言的产品可标注为"不适用"，但建议预留国际化扩展空间。

## 文案管理（MUST）

### 基本原则
1. 所有面向用户的文案必须使用国际化函数包裹，禁止硬编码。
2. 文案 key 使用英文命名空间 + 语义化路径。
3. 默认语言：简体中文（zh-CN），基础翻译语言：英文（en）。

### 文案 Key 命名规范

```
{module}.{page/component}.{element}.{action/description}

示例：
order.list.title               → "订单列表"
order.create.submit_button      → "提交订单"
order.create.success_message    → "订单创建成功"
common.action.confirm           → "确认"
common.action.cancel            → "取消"
common.error.network            → "网络异常，请稍后重试"
```

### 文案文件组织

```
locales/
├── zh-CN/
│   ├── common.json        ← 通用文案（按钮、状态、错误）
│   ├── order.json         ← 订单模块
│   ├── user.json          ← 用户模块
│   └── ...
├── en/
│   ├── common.json
│   ├── order.json
│   ├── user.json
│   └── ...
└── {locale}/
    └── ...
```

### 文案编写规则
1. 避免拼接字符串构造句子（不同语言语序不同）。
2. 使用参数化占位符：`"欢迎回来，{name}"` 而非 `"欢迎回来，" + name`。
3. 处理复数形式：使用 ICU MessageFormat 或框架内置的复数支持。
4. 避免在文案中包含 HTML 标签（使用富文本组件代替）。

## 各平台实现（MUST）

### Web 前端

| 框架 | 推荐方案 | 文案格式 |
|------|---------|---------|
| Vue 3 | vue-i18n | JSON |
| React | react-intl / i18next | JSON |
| 小程序 | 自定义 i18n 工具或 miniprogram-i18n | JSON |

### 移动端

| 平台 | 推荐方案 | 文案格式 |
|------|---------|---------|
| iOS | NSLocalizedString / String Catalog | .strings / .xcstrings |
| Android | Android Resources | strings.xml |
| Flutter | intl / easy_localization | ARB / JSON |

### 服务端

| 平台 | 说明 |
|------|------|
| Go | 错误消息使用错误码 + 客户端翻译，服务端不做文案翻译 |
| .NET | 同上，或使用 ResourceManager（面向管理后台的服务端渲染场景） |

### 服务端国际化策略
1. API 通过 `Accept-Language` 请求头传递客户端语言偏好。
2. 服务端返回错误码（如 `ERR_ORDER_NOT_FOUND`），客户端根据错误码显示本地化消息。
3. 服务端不负责翻译用户界面文案（翻译逻辑在客户端）。
4. 服务端负责的国际化：邮件模板、短信模板、推送通知（使用模板引擎 + 多语言模板）。

## 日期、时间与时区（MUST）

### 存储
1. 数据库和 API 中统一使用 UTC 时间存储。
2. 时间戳格式：ISO 8601（`2026-03-24T10:30:00Z`）。
3. 禁止使用本地时间存储（`2026-03-24 18:30:00+08:00` 仅用于展示）。

### 展示
1. 客户端根据用户时区偏好转换为本地时间展示。
2. 时区偏好来源优先级：用户设置 > 浏览器/系统时区。
3. 日期格式使用 `Intl.DateTimeFormat`（Web）或平台原生格式化 API。

### 日期格式参考

| 语言 | 短日期 | 长日期 | 时间 |
|------|--------|--------|------|
| zh-CN | 2026/03/24 | 2026年3月24日 | 18:30 |
| en-US | 03/24/2026 | March 24, 2026 | 6:30 PM |
| ja-JP | 2026/03/24 | 2026年3月24日 | 18:30 |

## 数字与货币（MUST）

### 数字格式
1. 使用 `Intl.NumberFormat`（Web）或平台原生格式化 API。
2. 不硬编码千分位分隔符和小数点符号。

| 语言 | 数字格式 | 百分比 |
|------|---------|--------|
| zh-CN | 1,234,567.89 | 85.5% |
| en-US | 1,234,567.89 | 85.5% |
| de-DE | 1.234.567,89 | 85,5 % |

### 货币格式
1. 货币金额必须携带货币代码（ISO 4217）：`CNY`、`USD`、`EUR`。
2. 存储使用最小单位整数（如人民币用分、美元用美分），防止浮点精度问题。
3. 展示时使用本地化货币格式。

| 语言 | 人民币 | 美元 |
|------|--------|------|
| zh-CN | ¥1,234.56 | US$1,234.56 |
| en-US | CN¥1,234.56 | $1,234.56 |

## RTL（从右到左）支持（SHOULD）

### 适用语言
- 阿拉伯语（ar）、希伯来语（he）、波斯语（fa）、乌尔都语（ur）。

### RTL 实现要点
1. HTML 根元素设置 `dir="rtl"`。
2. CSS 使用逻辑属性：`margin-inline-start` 代替 `margin-left`。
3. 图标/图片：方向性图标需镜像（箭头、进度条），非方向性图标不镜像（logo、照片）。
4. 布局：Flexbox 和 Grid 的 `start` / `end` 自动适配 RTL。

## 国际化测试（SHOULD）

### 测试清单
1. **伪本地化测试**：使用加长字符串（如 `[!!确认!!]`）验证 UI 是否溢出。
2. **多语言切换测试**：运行时切换语言，UI 完整更新。
3. **日期/时间/货币测试**：不同语言环境下格式正确。
4. **文本方向测试**（如支持 RTL）：布局正确镜像。
5. **缺失翻译降级**：缺少翻译时显示默认语言文案（不显示 key）。
