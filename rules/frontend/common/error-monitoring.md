# rules/frontend/common/error-monitoring.md

## 文档目标
1. 定义三端共用的前端异常捕获、错误上报、行为埋点规范。
2. 具体框架层的错误边界参见 `frameworks/*.md`；API 错误码映射参见 `common/baseline.md`。

---

## 异常捕获（MUST）

### 全局捕获
1. 必须注册全局错误捕获机制，防止未处理异常导致白屏：
   - **Vue 项目**：`app.config.errorHandler` 捕获组件渲染错误。
   - **React 项目**：`ErrorBoundary` 组件包裹页面级路由。
   - **所有项目**：`window.onerror` 捕获未被框架拦截的运行时错误。
   - **所有项目**：`window.addEventListener('unhandledrejection', ...)` 捕获未处理的 Promise 拒绝。
2. 小程序项目必须注册 `App.onError` 和 `App.onUnhandledRejection`。
3. 全局捕获必须在应用初始化阶段注册，禁止延迟注册导致早期错误遗漏。

### 错误分类
1. 异常必须分类上报，至少区分以下类型：
   - **JS 运行时错误**：类型错误、引用错误、语法错误等。
   - **资源加载错误**：脚本、样式、图片加载失败。
   - **接口错误**：请求超时、HTTP 状态码异常、业务错误码。
   - **框架错误**：组件渲染崩溃（Vue errorHandler / React ErrorBoundary）。
2. 每条错误上报必须包含：错误类型、错误消息、堆栈信息、页面路径、用户标识（脱敏）、时间戳、设备/浏览器信息。

检查方式：代码审查
阻断级别：阻断合并

## 错误上报（MUST）

1. 项目必须接入统一的错误监控平台（如 Sentry、Fundebug 或自建平台）。
2. 上报内容必须包含 Source Map 映射，确保生产环境压缩代码可还原到源码位置。
3. Source Map 文件禁止部署到生产 CDN，仅上传到监控平台或私有存储。
4. 相同错误必须聚合（按堆栈指纹去重），避免告警风暴。
5. 上报频率必须有限流策略（如同一错误每分钟最多上报 10 次），防止客户端循环错误导致上报洪泛。

### 告警规则（MUST）
1. 新增错误类型必须触发告警。
2. 错误量突增（如 5 分钟内错误数 > 基线 3 倍）必须触发告警。
3. 关键页面（首页、支付、登录）的错误必须设置独立告警规则，响应时间更短。

检查方式：监控平台配置审查
阻断级别：阻断合并

## 接口错误监控（MUST）

1. 统一请求层（Axios 拦截器 / uni.request 适配层）必须自动上报接口异常：
   - HTTP 状态码 ≥ 400 的响应。
   - 请求超时。
   - 网络连接失败。
2. 上报内容必须包含：接口路径、请求方法、状态码、错误码、耗时、请求标识（`request_id`）。
3. 敏感信息（Token、密码、证件号）禁止出现在上报数据中，必须脱敏。
4. 核心接口（登录、支付、下单）的成功率必须纳入监控看板。

检查方式：代码审查 + 监控看板审查
阻断级别：阻断合并

## 性能埋点（SHOULD）

1. 核心页面接入 Web Vitals 上报（LCP、FID、CLS），持续跟踪性能趋势。
2. 小程序接入性能上报（首屏时间、setData 耗时）。
3. 关键用户操作（页面跳转、表单提交、列表加载）记录耗时埋点。
4. 埋点数据采样率可配置（建议 10%-100%），避免影响用户体验。

检查方式：监控看板审查
阻断级别：告警记录

---

## 用户行为埋点（SHOULD）

1. 页面浏览（PV/UV）、按钮点击、关键业务流程必须有行为埋点。
2. 埋点方案推荐"声明式 + 自动化"：
   - **自动埋点**：页面浏览、路由切换由框架自动上报。
   - **手动埋点**：业务关键动作（下单、支付、分享）手动调用上报函数。
3. 埋点 SDK 必须统一封装，业务代码通过统一 API 调用，禁止直接引用第三方 SDK 内部接口。
4. 埋点数据禁止包含用户隐私信息（手机号、身份证等），必须脱敏或匿名化。

检查方式：代码审查
阻断级别：告警记录

---

## 监控平台接入配置（MUST）

1. 监控 SDK 初始化必须在应用入口处完成（`main.ts` / `App.vue` / `App.tsx`）。
2. 不同环境（开发/测试/生产）必须使用不同的上报 DSN / 项目标识，禁止开发环境数据污染生产监控。
3. 开发环境可关闭自动上报（仅控制台输出），但必须保留捕获逻辑以便本地调试。
4. 监控 SDK 版本必须锁定，升级需单独提交并验证兼容性。

检查方式：配置审查
阻断级别：阻断合并
