# rules/frontend/common/normalization.md

## 文档目标
1. 统一 UI/样式规范化改造流程，保证“外观稳定 + 规则收敛”。
2. 在不改变业务行为的前提下，逐步收敛到 token 与统一组件体系。

## 适用场景
1. 页面重构、组件替换、样式治理。
2. 老页面规范化接入（Tailwind/UnoCSS、token、适配层）。

## 改造原则（MUST）
1. 最小改动原则：不改变视觉和业务行为。
2. 优先复用原则：先复用现有组件与 token，再新增能力。
3. 可回滚原则：每轮改造应可独立回滚。
检查方式：人工审查  
阻断级别：告警记录

## Token 优先规则（MUST）
1. 颜色、字号、间距、圆角等设计值必须优先使用 token。
2. 发现硬编码样式值时，先评估复用 token，再决定新增 token。
3. 新增 token 必须采用语义命名，不得使用页面或业务命名。
检查方式：人工审查 + 静态扫描  
阻断级别：阻断合并

## `<style>` 使用规则（uni-app）
1. `SHOULD`：页面与业务组件优先使用 `UnoCSS` 原子类，不写 `<style>`。
2. `MUST`：必须写 `<style>` 时，样式块顶部写中文原因说明。
3. `MUST`：`<style>` 内禁止硬编码主题色和尺寸，需引用 token 或语义变量。
检查方式：人工审查 + 静态扫描  
阻断级别：告警记录

## tab-host 规范化规则（适用时 MUST）
1. 多 Tab 同页切换场景优先采用 `tab-host` 宿主页模式。
2. `tab-host` 负责状态与切换，Tab 子视图只负责业务内容渲染。
3. 不适用 `tab-host` 时，必须在评审说明替代方案与理由。
检查方式：人工审查  
阻断级别：告警记录

## 小程序专项规则（MUST）
1. 图标仅允许位图（`png/jpg/jpeg/webp`），禁止 `svg` 与 emoji 图标替代。
2. 主包超过 `2MB` 必须拆分分包后再发版。
3. 平台能力调用必须收敛到 `platform/mp-weixin` 适配层。
检查方式：静态扫描 + CI 阻断  
阻断级别：阻断合并
