# 微信 H5 工具包模板

## 文档目标
1. 提供机型与系统兼容测试清单、H5 活动场景目录归档规则。
2. 微信 H5 规则参见 `applications/wechat-h5.md`。

## 使用方式
- **谁用**：微信 H5 开发者、测试人员。
- **何时用**：H5 页面提测前进行兼容测试；活动 H5 开发时按归档规则组织目录。
- **怎么用**：兼容测试清单用于提测/上线前逐项确认；活动目录规则在项目初始化时遵守。

## 二、H5 活动场景目录归档规则

### 2.1 问题背景
- 活动 H5 通常是短期项目，生命周期短（几天到几周）
- 不归档会导致活动代码散落各处，难以查找和维护
- 过期活动代码不清理会增加构建体积和维护负担

### 2.2 目录结构规则

```
src/
  views/
    activities/              ← 所有活动页面统一归入此目录
      _archived/             ← 已下线活动归档（构建时排除）
        2024-spring-sale/
        2024-618/
      2025-new-year/         ← 当前活动（按活动标识命名）
        index.vue            ← 活动主页面
        components/          ← 活动私有组件
          PrizeWheel.vue
          CouponCard.vue
        composables/         ← 活动私有逻辑
          useActivity.ts
        config.ts            ← 活动配置（时间、规则、奖品等）
        types.ts             ← 活动类型定义
      2025-spring-sale/
        index.vue
        ...
```

### 2.3 命名规则（MUST）

| 规则 | 说明 | 示例 |
|------|------|------|
| 目录名格式 | `{年份}-{活动标识}` | `2025-new-year`、`2025-618` |
| kebab-case | 全小写，单词用 `-` 连接 | `2025-spring-sale`（非 `2025SpringSale`） |
| 语义化 | 使用活动名而非项目编号 | `2025-mid-autumn`（非 `project-123`） |
| 活动配置外置 | 时间/规则/奖品等写入 `config.ts` | 便于快速修改，无需改动组件代码 |

### 2.4 活动生命周期管理

```
1. 创建活动目录
   └─ 在 activities/ 下新建 {年份}-{活动标识}/ 目录
       │
       ▼
2. 开发与上线
   └─ 活动代码仅在活动目录内，不污染公共模块
       │
       ▼
3. 活动下线（MUST）
   └─ 下线后 1 周内将目录移入 _archived/
       │
       ▼
4. 归档处理（MUST）
   ├─ 路由配置中移除活动页面
   ├─ _archived/ 目录在构建配置中排除
   └─ 确认无其他页面引用活动组件
       │
       ▼
5. 定期清理（SHOULD）
   └─ 每季度评估 _archived/ 中超过 6 个月的活动
      └─ 确认无复用价值后删除
```

### 2.6 活动配置模板

```ts
// activities/2025-new-year/config.ts
// 活动配置：时间、规则、资源等集中管理

export const ACTIVITY_CONFIG = {
  /** 活动标识 */
  id: '2025-new-year',
  /** 活动名称 */
  name: '2025 新年活动',

  /** 活动时间 */
  startTime: '2025-01-01T00:00:00+08:00',
  endTime: '2025-01-31T23:59:59+08:00',

  /** 活动状态判断 */
  get isActive(): boolean {
    const now = Date.now()
    return now >= new Date(this.startTime).getTime()
      && now <= new Date(this.endTime).getTime()
  },

  /** 分享配置 */
  share: {
    title: '新年活动来啦',
    desc: '参与活动赢好礼',
    imgUrl: 'https://cdn.example.com/activities/2025-new-year/share.png',
  },
} as const
```

## 约束清单

| 编号 | 级别 | 规则 |
|------|------|------|
| 1 | MUST | H5 页面上线前按兼容测试清单完成核心机型验证 |
| 2 | MUST | 活动 H5 统一归入 `views/activities/` 目录 |
| 3 | MUST | 活动目录命名格式为 `{年份}-{活动标识}`（kebab-case） |
| 4 | MUST | 活动下线后 1 周内移入 `_archived/` 并移除路由注册 |
| 5 | MUST | `_archived/` 目录在构建时排除，不打入产物 |
| 6 | SHOULD | 活动配置（时间/规则/奖品）外置到 `config.ts`，不硬编码在组件中 |
| 7 | SHOULD | 每季度清理超过 6 个月的归档活动代码 |

检查方式：人工审查 + 目录结构扫描
阻断级别：MUST 条款阻断合并
