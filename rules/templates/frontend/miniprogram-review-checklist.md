# 小程序审核清单模板

## 文档目标
1. 定义微信小程序提审前的自查清单，减少审核驳回。
2. 平台规则约束参见 `applications/miniprogram.md`。

---

## 使用方式
1. 每次提审前由开发者逐项自查，全部 P0 通过后方可提审。
2. 每项给出结果：`[x]` 通过 / `[ ]` 不通过（需写原因和修复方案）。
3. 自查记录保留在提审 PR 或发版记录中，便于追溯。

## 优先级说明
1. `P0`：审核高频驳回项，必须全部通过后提审。
2. `P1`：改进项，允许带条件提审但必须登记后续修复计划。

---

## 基本信息
- [ ] [P0] 已确认提审版本号与构建产物一致
- [ ] [P0] 已确认小程序名称、简介、类目与实际功能匹配
- [ ] [P0] 已确认隐私政策弹窗在首次使用时展示，且用户可拒绝

## 包体积
- [ ] [P0] 主包体积 ≤ 2MB
- [ ] [P0] 单个分包体积 ≤ 2MB
- [ ] [P0] 总包体积 ≤ 20MB
- [ ] [P1] 图片资源已压缩，大图已上传 CDN 而非打包进代码

## 内容合规
- [ ] [P0] 无硬编码测试数据、假数据、占位文案（如"测试商品"、"Lorem ipsum"）
- [ ] [P0] 无未脱敏的手机号、姓名、地址等个人信息
- [ ] [P0] 无涉政、涉黄、涉赌等违规内容
- [ ] [P0] 用户生成内容（UGC）场景有内容审核机制或报告入口
- [ ] [P0] 无指向外部 App 下载或非微信生态的引导（如"请到 App Store 下载"）

## 功能完整性
- [ ] [P0] 所有页面可正常访问，无空白页、死链、404 页面
- [ ] [P0] 核心业务流程（注册/登录/下单/支付）完整可走通
- [ ] [P0] 登录功能支持游客模式或明确告知登录原因，非强制登录才能浏览
- [ ] [P0] 授权拒绝后不阻塞使用，有明确引导重新授权的路径
- [ ] [P1] 列表页空态有友好提示，非空白展示

## 权限与隐私
- [ ] [P0] 仅申请业务必需的权限（位置、相机、相册、通讯录等）
- [ ] [P0] 权限申请时机合理（用到时才弹窗，非启动时批量申请）
- [ ] [P0] `app.json` 中 `permission` 字段的用途描述真实准确
- [ ] [P0] 隐私协议中列出的数据收集项与实际一致
- [ ] [P0] `__usePrivacyCheck__` 已开启（2023 年 9 月后要求）

## 用户体验
- [ ] [P0] 页面加载超过 1 秒有 loading 或骨架屏反馈
- [ ] [P0] 网络异常有明确提示和重试入口，非空白或卡死
- [ ] [P0] 支付失败、接口报错等异常场景有用户可理解的提示
- [ ] [P1] 页面间跳转流畅，无明显卡顿或闪烁
- [ ] [P1] 深色模式（如已适配）下文字和背景对比度正常

## 资源与素材
- [ ] [P0] 图标资源使用 `png/jpg/jpeg` 格式，无 SVG（小程序不支持）
- [ ] [P0] 无侵权素材（字体、图片、图标需确认授权）
- [ ] [P0] 小程序图标清晰，无拉伸变形，符合微信设计规范

## 分享与社交
- [ ] [P0] 分享卡片标题和图片与页面内容匹配，无误导
- [ ] [P0] 分享链接点击后可正常打开对应页面
- [ ] [P1] 分享图片尺寸符合要求（5:4 比例，建议 500×400px）

## 支付（涉及支付功能时）
- [ ] [P0] 支付金额与展示金额一致
- [ ] [P0] 支付前有订单确认页，用户明确知道支付内容和金额
- [ ] [P0] 支付结果页正确展示成功/失败/待确认状态
- [ ] [P0] 退款流程说明清晰可见

## 接口与安全
- [ ] [P0] 所有接口使用 HTTPS，无 HTTP 明文请求
- [ ] [P0] 接口域名已在小程序管理后台「服务器域名」中配置
- [ ] [P0] 无硬编码密钥、token、测试环境地址
- [ ] [P0] `request` 合法域名、`uploadFile` 域名、`downloadFile` 域名均已配置

## 发布配置
- [ ] [P0] 已关闭调试模式（`enableDebug: false`）
- [ ] [P0] 已确认 `app.json` 中 `pages` 首项为正确的首页
- [ ] [P0] 版本描述填写了本次更新内容摘要
- [ ] [P1] 已配置灰度发布比例（建议先 10% → 50% → 全量）

---

## 结论
- [ ] `提审`（全部 P0 通过）
- [ ] `暂缓`（存在 P0 未通过，需修复后重新自查）
- [ ] `有条件提审`（P0 通过，P1 未通过已登记修复计划）
