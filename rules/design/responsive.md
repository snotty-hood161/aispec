# rules/design/responsive.md — 响应式设计规范

## 目的
定义多端适配的设计策略、断点标准和布局变化规则，确保产品在不同设备上均有良好体验。

## 断点定义

| 断点名称 | 宽度范围 | 典型设备 | Token 名 |
|---------|---------|---------|---------|
| xs | < 480px | 小屏手机 | `breakpoint-xs` |
| sm | 480px - 767px | 大屏手机 | `breakpoint-sm` |
| md | 768px - 1023px | 平板竖屏 | `breakpoint-md` |
| lg | 1024px - 1439px | 平板横屏/小笔记本 | `breakpoint-lg` |
| xl | 1440px - 1919px | 桌面 | `breakpoint-xl` |
| 2xl | ≥ 1920px | 大屏桌面 | `breakpoint-2xl` |

### 断点使用原则
- **移动优先**：以最小断点为基础编写样式，逐步向大屏增强。
- **内容驱动**：断点选择应基于内容何时"断裂"，而非固定设备宽度。
- **最少断点**：大多数项目使用 3 个断点即可（sm/md/xl）。

## 布局策略

### 导航适配

| 端 | 导航模式 | 说明 |
|----|---------|------|
| 桌面（≥1024px） | 侧边导航展开 或 顶部导航 | 完整显示所有导航项 |
| 平板（768-1023px） | 侧边导航折叠 或 顶部导航 | 图标导航，悬停展开标签 |
| 手机（<768px） | 底部标签栏 + 汉堡菜单 | 底部放 3-5 核心入口，其余收入汉堡菜单 |

### 内容区域适配

| 端 | 栏数 | 内容策略 |
|----|------|---------|
| 桌面 | 12 栏栅格 | 多列布局，侧边信息面板 |
| 平板 | 8 栏栅格 | 减少列数，关键信息保留 |
| 手机 | 4 栏栅格 | 单列布局，信息纵向堆叠 |

### 组件适配规则

| 组件 | 桌面 | 平板 | 手机 |
|------|------|------|------|
| 表格 | 完整表格 | 可滚动表格/减少列 | 卡片列表替代 |
| 卡片网格 | 3-4 列 | 2-3 列 | 1-2 列 |
| 对话框 | 居中弹窗（max-width: 640px） | 居中弹窗 | 全屏/底部抽屉 |
| 表单 | 多列布局 | 多列布局 | 单列全宽 |
| 操作栏 | 行内按钮 | 行内按钮 | 固定底部操作栏 |
| 分页 | 完整分页器 | 简洁分页 | 加载更多按钮 |
| 搜索 | 展开搜索框 | 展开搜索框 | 图标点击展开 |
| 日期选择 | 日历弹窗 | 日历弹窗 | 原生日期选择器 |

## 触控适配

### 手机端专用规则
- 触控目标最小 44x44px。
- 关键操作放在屏幕下半部拇指热区。
- 避免精确的拖拽操作，改用长按或滑动。
- 列表项支持左右滑动操作（如滑动删除）。
- 下拉刷新 + 上拉加载更多。
- 避免 hover 效果（触屏无悬停）。

### 平板端专用规则
- 同时支持触控和键盘/鼠标操作。
- 横屏和竖屏均需适配。
- 可使用分屏布局（主从结构）。

## 图片与资源适配

### 图片策略
- 提供 1x、2x 分辨率（`srcset`）。
- 使用 WebP 格式（降级 PNG/JPG）。
- 设置合理的图片宽度约束（`max-width: 100%`）。
- 大图使用懒加载。

### 字号适配
- 手机端正文最小 16px（避免 iOS 自动放大）。
- 标题在手机端适当缩小（但保持层级关系）。
- 使用 `clamp()` 实现流式字号：
  ```css
  font-size: clamp(1.5rem, 2vw + 1rem, 2.5rem);
  ```

## 设计交付要求

### 需要设计的断点
- **最少交付**：桌面（1440px）+ 手机（375px）。
- **推荐交付**：桌面（1440px）+ 平板（768px）+ 手机（375px）。
- **完整交付**：桌面（1440px）+ 大屏（1920px）+ 平板（768px）+ 手机（375px）。

### 标注规范
- 在设计稿中标注各断点下的布局变化。
- 标注隐藏/显示的元素。
- 标注变化的间距和字号。
- 标注交互方式的变化（如悬停 → 长按）。
