# rules/design/accessibility.md — 无障碍设计规范

## 目的
确保设计产出满足无障碍标准，让所有用户（包括视觉、听觉、运动或认知障碍用户）都能使用产品。

## 适用级别
- **MUST**（必须）标记的条款为强制要求，对应 WCAG 2.1 AA 级。
- **SHOULD**（应该）标记的条款为推荐实践，对应 WCAG 2.1 AAA 级。

## 色彩对比度

### MUST — AA 级标准
- 正文文本与背景的对比度 ≥ **4.5:1**。
- 大号文本（≥18px 粗体或 ≥24px 常规）与背景对比度 ≥ **3:1**。
- UI 组件和图形元素与相邻颜色的对比度 ≥ **3:1**。
- 禁用状态的元素不受对比度要求约束（已通过视觉样式传达"不可用"）。

### SHOULD — AAA 级标准
- 正文文本与背景的对比度 ≥ **7:1**。
- 大号文本与背景对比度 ≥ **4.5:1**。

### 色彩使用规则
- **MUST** — 颜色不能是传达信息的唯一手段。
  - 错误状态：红色 + 错误图标 + 错误文字。
  - 成功状态：绿色 + 勾选图标 + 成功文字。
  - 图表数据：颜色 + 图案/形状区分。
- **MUST** — 链接文字与普通文字除颜色外，还有其他视觉区分（如下划线）。

## 触控与交互

### MUST — 触控目标
- 可点击元素的最小尺寸为 **44 × 44 px**（CSS 像素）。
- 相邻触控目标之间至少有 **8px** 间距。
- 如果视觉尺寸小于 44px（如图标按钮），使用 padding 扩展点击区域。

### MUST — 键盘可访问
- 所有可交互元素可通过 Tab 键聚焦。
- 聚焦状态有清晰可见的焦点环（focus ring）。
- 弹窗/抽屉打开时焦点陷阱（focus trap）在弹窗内。
- ESC 键可关闭弹窗/下拉菜单。
- Enter/Space 可激活按钮和链接。

### SHOULD — 增强交互
- 提供键盘快捷键用于高频操作。
- 拖拽操作提供键盘替代方案。
- 自定义组件（如日期选择器）支持方向键导航。

## 文本与内容

### MUST — 文本可读性
- 文本可缩放至 200% 而不丢失内容或功能。
- 行高 ≥ 1.5 倍字号（段落文本）。
- 段间距 ≥ 字号的 2 倍。
- 正文不使用两端对齐（中文例外）。

### MUST — 图片与媒体
- 功能性图片提供 alt 文本。
- 装饰性图片使用空 alt（`alt=""`）。
- 视频提供字幕（如适用）。
- 音频提供文字稿（如适用）。

### SHOULD — 语言与认知
- 使用简洁清晰的语言。
- 避免专业术语（或提供解释）。
- 错误消息说明原因和解决方法。
- 重要操作提供帮助文本或工具提示。

## 动效与动画

### MUST — 动效安全
- 尊重系统的 `prefers-reduced-motion` 设置。
- 不使用闪烁频率 3 次/秒以上的动画（可能诱发光敏性癫痫）。
- 自动播放的动画提供暂停/停止按钮。

### SHOULD — 动效体验
- 动画持续时间不超过 5 秒（循环动画除外）。
- 关键信息不仅依赖动画传达。
- 页面过渡动画不阻塞用户操作。

## 表单无障碍

### MUST — 表单标签
- 每个输入控件有关联的标签（`<label>` 或 `aria-label`）。
- 必填字段有视觉和语义标记（`*` + `required`/`aria-required`）。
- 表单组使用 `fieldset` + `legend` 分组。
- 错误信息与对应输入框关联（`aria-describedby`）。

### MUST — 表单交互
- 错误信息在提交时自动定位到第一个错误字段。
- 校验结果支持屏幕阅读器朗读。
- 密码输入提供"显示/隐藏"切换。

## 设计审查清单

在设计验证阶段（Phase 4），以下清单必须逐项检查：

- [ ] 所有文本对比度 ≥ 4.5:1（大号文本 ≥ 3:1）
- [ ] 颜色不是唯一信息传达手段
- [ ] 所有可交互元素 ≥ 44x44px
- [ ] 焦点状态可见
- [ ] 弹窗/浮层有键盘关闭方式
- [ ] 图片有适当的 alt 属性策略
- [ ] 动画尊重 `prefers-reduced-motion`
- [ ] 表单标签完整关联
- [ ] 错误状态有清晰的视觉和文字提示
- [ ] 文本可缩放至 200% 不丢失功能
