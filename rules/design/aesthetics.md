# rules/design/aesthetics.md — 视觉美学指南

## 目的
指导 AI 在设计 UI 时做出有品位、有个性、有记忆点的视觉决策。不是限制创意，而是避免平庸。

## 设计前准备 — 确定审美方向

在开始任何视觉设计之前，必须先理解上下文并承诺一个**大胆的审美方向**：

### 1. 理解产品上下文
- **目的**：这个界面解决什么问题？谁在使用它？
- **调性**：产品给人的第一感觉应该是什么？
- **约束**：技术框架、性能要求、无障碍标准。

### 2. 选择审美方向
从以下方向中选择或组合（仅为灵感启发，可以创造新的方向）：

| 方向 | 特征 | 适用场景 |
|------|------|---------|
| 极简克制 | 大量留白、精确排版、细节取胜 | 工具类、效率类产品 |
| 丰富表现 | 色彩鲜明、动效丰富、视觉冲击 | 创意类、娱乐类产品 |
| 专业严谨 | 高信息密度、数据可视化、功能导向 | 企业级、B2B 产品 |
| 温暖亲和 | 圆润造型、柔和配色、引导性强 | 教育类、社交类产品 |
| 高端精致 | 克制用色、精良排版、质感细节 | 品牌官网、高端产品 |
| 活力潮流 | 大胆配色、创意布局、年轻化 | 年轻用户、潮流产品 |
| 复古怀旧 | 经典元素、怀旧配色、温情质感 | 文化类、情怀类产品 |
| 未来科技 | 深色主题、霓虹色彩、科技感 | 技术类、数据类产品 |
| 编辑杂志 | 强排版、大标题、图文混排 | 内容类、媒体类产品 |
| 粗野主义 | 原始粗犷、高对比、打破常规 | 实验性、独立项目 |
| 有机自然 | 不规则形状、自然纹理、柔和过渡 | 健康类、环保类产品 |
| 装饰艺术 | 几何对称、金色点缀、精致线条 | 奢侈品、高端服务 |

### 3. 定义设计差异化
回答一个关键问题：**这个设计让人记住的一个点是什么？**

可以是：
- 一个独特的配色方案
- 一个意想不到的布局方式
- 一个精心设计的微交互
- 一种独特的字体组合
- 一个有创意的视觉元素

**关键原则：大胆的极简主义和精致的极繁主义都是好设计——关键是意图明确，而不是强度高低。**

## 字体排印

### 字体选择原则
- **选择有特色、独特、有美感的字体**。避免 Arial、Inter、Roboto 等无个性字体。
- **展示字体与正文字体配对**：用有特色的展示字体做标题，用精致的正文字体做内容。
- 中文场景推荐搭配：
  - 展示字体：思源宋体、方正标雅宋、阿里巴巴普惠体 Heavy、汉仪旗黑
  - 正文字体：思源黑体、苹方、鸿蒙字体（HarmonyOS Sans）
  - 等宽字体：JetBrains Mono、Fira Code、Source Code Pro
- 英文/混排推荐搭配（优先从 Google Fonts 引入）：
  - 展示：Playfair Display、DM Serif Display、Fraunces、Sora、Outfit
  - 正文：DM Sans、Plus Jakarta Sans、Satoshi、General Sans
  - 避免：Inter、Roboto、Arial、Space Grotesk、Open Sans

### 字号层级
- 标题必须有明确的视觉层级（至少 3 级）。
- 正文字号不小于 14px（移动端不小于 16px）。
- 行高 1.4-1.6（中文偏 1.6，英文偏 1.4）。

### 排版细节
- 字间距：中文 0-0.05em，英文标题可适当加宽（0.02-0.05em）。
- 段间距：至少为行高的 0.5 倍。
- 最大行宽：正文不超过 70 个字符（中文约 35 字）。

## 色彩与主题

### 配色原则
- **承诺一个完整的配色体系**，用 CSS 变量保持一致性。
- **主色 + 强调色 > 平均分配**：1-2 个主导色加 1 个锐利的强调色，效果远好于 4-5 个平均使用的颜色。
- **避免 AI 生成的万能配色**：特别禁止紫色渐变白底、千篇一律的蓝紫色系。

### 配色检查清单
- [ ] 配色方案是否与产品调性匹配？
- [ ] 主色是否有足够的辨识度？
- [ ] 强调色是否与主色形成有效对比？
- [ ] 深浅模式切换时，配色是否仍然和谐？
- [ ] 配色是否满足 WCAG AA 对比度标准？

### 主题设计
- 浅色模式：背景不要纯白（`#FFFFFF`），使用带微微色调的白（如 `#FAFAFA`、`#F8F9FB`）。
- 深色模式：背景不要纯黑（`#000000`），使用深灰（如 `#0A0A0A`、`#111111`、`#1A1A1A`）。
- 在不同生成中交替使用浅色和深色方案，避免总是收敛到同一种。

## 空间构图

### 布局原则
- **拥抱非对称**：不是所有内容都需要居中。
- **有意的重叠**：元素间的有意重叠可以创造层次感。
- **对角线流动**：视线引导可以使用对角线而不仅是垂直/水平。
- **打破栅格**：偶尔突破栅格系统的元素可以创造视觉焦点。
- **有态度的留白**：大量留白 OR 高密度布局都可以——关键是有意识的选择。

### 避免的布局模式
- 所有页面都是居中的卡片堆叠。
- 所有区域都是等宽等高的网格。
- 没有视觉焦点的均匀分布。
- 缺乏呼吸感的拥挤排列。

## 背景与视觉细节

### 创造氛围与深度
不要默认使用纯色背景。根据审美方向选择合适的背景处理：

| 技法 | 效果 | 适用场景 |
|------|------|---------|
| 渐变网格 | 柔和、现代、有层次感 | 着陆页、创意页 |
| 噪点纹理 | 质感、复古、有触感 | 文艺类、品牌类 |
| 几何图案 | 有序、精致、专业感 | 企业级、工具类 |
| 层叠透明 | 深度感、空间感 | 仪表盘、数据类 |
| 明暗对比 | 聚焦、高级感 | 高端品牌、产品展示 |
| 装饰边框 | 个性、有界感 | 独立项目、个人品牌 |
| 颗粒叠加 | 胶片感、有温度 | 社交类、生活类 |

### 视觉细节加分项
- 自定义光标样式（如品牌光标）。
- 有趣的空状态插画/动画。
- 滚动视差效果。
- 悬停时的微动效。
- 加载时的骨架屏或品牌化加载动画。

## 动效与微交互

### 动效策略
- **优先 CSS 原生方案**：`transition`、`animation`、`@keyframes`。
- **React 项目**：优先使用 Framer Motion。
- **Vue 项目**：优先使用 `<Transition>` 和 `<TransitionGroup>`。
- **聚焦高影响力时刻**：一个精心编排的页面加载动画（交错入场 `animation-delay`），比散落的微交互更令人愉悦。

### 推荐的动效场景
1. **页面加载** — 元素交错入场，从主要到次要依次显现。
2. **滚动触发** — 元素进入视口时的渐显/上移效果。
3. **悬停状态** — 出乎意料但不喧宾夺主的悬停反馈。
4. **状态切换** — 开关、标签页、折叠面板的平滑过渡。
5. **数据更新** — 数字变化的过渡动画、列表项的增删动效。

### 动效参数参考
| 场景 | 时长 | 缓动函数 |
|------|------|---------|
| 小元素状态变化 | 150-200ms | ease-out |
| 常规过渡 | 250-350ms | ease-in-out |
| 页面级过渡 | 400-600ms | cubic-bezier(0.4, 0, 0.2, 1) |
| 弹性效果 | 500-800ms | cubic-bezier(0.68, -0.55, 0.265, 1.55) |

## 实施复杂度匹配

**重要原则：实施复杂度必须匹配审美愿景。**

- **极繁主义设计** — 需要精心编写的大量动画和特效代码，多层叠加效果，丰富的状态变化。
- **极简主义设计** — 需要精准到像素的克制与节制，每一个间距、字号、色值都经过反复推敲，优雅来自于完美地执行简单。

两种方向都需要同等的专注度和代码质量，区别仅在于表达方式。

## 多样性要求

**每次设计都应该有独特性。** 在不同项目之间必须变化：
- 在浅色和深色主题间交替。
- 使用不同的字体组合。
- 尝试不同的审美方向。
- 避免在不同项目中收敛到同一种"安全"的风格。
