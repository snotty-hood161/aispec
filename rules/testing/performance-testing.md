# 性能测试规范

## Skill 协作
1. QA 任务优先使用 `$qa-test-strategist`，自动加载本规则。
2. DevOps 任务使用 `$devops-engineer`，关注性能基线和 SLO 定义。
3. 跨域业务任务使用 `$task-router` 自动路由。

## 适用范围
1. 核心 API 和关键业务流程必须定义性能基线。
2. 上线前必须完成至少一轮性能基线测试。
3. 重大功能变更或架构调整后必须回归性能测试。

## 性能测试类型（MUST）

| 类型 | 目标 | 时机 | 持续时间 |
|------|------|------|---------|
| 基线测试 | 建立正常负载下的性能基准 | 首次上线前 | 10-30 分钟 |
| 负载测试 | 验证预期负载下系统表现 | 每次发布前 | 30-60 分钟 |
| 压力测试 | 找到系统瓶颈和崩溃点 | 重大更新前 | 60+ 分钟 |
| 浸泡测试 | 检测内存泄漏和资源耗尽 | 季度/重大更新 | 4-24 小时 |
| 峰值测试 | 验证突发流量的承受能力 | 大促/活动前 | 15-30 分钟 |

## 性能指标（MUST）

### 核心指标

| 指标 | 定义 | 采集方式 |
|------|------|---------|
| 响应时间（P50/P95/P99） | 请求从发送到收到响应的耗时 | 客户端测量 |
| 吞吐量（TPS/RPS） | 每秒处理的请求数 | 服务端测量 |
| 错误率 | 失败请求占总请求的比例 | 服务端测量 |
| 并发用户数 | 同时活跃的用户/连接数 | 测试工具统计 |

### 资源指标

| 指标 | 告警阈值 | 说明 |
|------|---------|------|
| CPU 使用率 | < 70%（常规），< 85%（峰值） | 超过则需要扩容或优化 |
| 内存使用率 | < 80% | 持续增长提示内存泄漏 |
| 数据库连接数 | < 连接池上限的 80% | 超过则调整连接池或优化查询 |
| 磁盘 I/O | < 80% 利用率 | 超过则需要存储优化 |

## 性能基线定义（MUST）

每个服务必须定义性能基线：

```markdown
## 服务性能基线

### 服务名称：{service-name}
### 基线日期：{yyyy-MM-dd}
### 测试环境：{环境配置描述}

### API 性能基线
| API 端点 | 方法 | P50 | P95 | P99 | TPS 上限 | 错误率上限 |
|---------|------|-----|-----|-----|---------|-----------|
| /api/v1/orders | GET | 50ms | 100ms | 200ms | 500 | 0.1% |
| /api/v1/orders | POST | 80ms | 150ms | 300ms | 200 | 0.1% |

### 资源基线（预期负载下）
| 指标 | 基线值 | 上限 |
|------|--------|------|
| CPU | 30% | 70% |
| 内存 | 512MB | 1GB |
| 数据库连接 | 20 | 50 |
```

## 测试场景设计（MUST）

### 场景模板

```
## 性能测试场景

### 场景名称：{场景名称}
### 测试目标：{验证什么}

### 用户行为模型
- 场景描述：{用户操作步骤}
- 思考时间：{步骤间等待时间}
- 数据参数化：{测试数据来源}

### 负载模型
- 起始并发：{n} 用户
- 递增策略：每 {n} 秒增加 {n} 用户
- 目标并发：{n} 用户
- 持续时间：{n} 分钟

### 通过标准
- P95 响应时间 < {n}ms
- 错误率 < {n}%
- TPS > {n}
```

### 必测场景
1. **登录流程** — 验证认证服务的并发承受能力。
2. **核心查询** — 列表查询（带分页/筛选/排序）在大数据量下的表现。
3. **核心写入** — 创建/更新核心业务对象的吞吐量。
4. **文件操作** — 文件上传/下载在并发下的表现。
5. **混合负载** — 模拟真实用户的混合操作（80% 读 + 20% 写）。

## CI 集成（SHOULD）

### 性能回归检测

```
性能测试在 CI 中的位置：
  → 构建完成
  → 部署到性能测试环境
  → 运行性能基线测试（k6 或等效工具）
  → 与基线对比
  → P95 退化 > 20% → 标记 Warning
  → P95 退化 > 50% → 阻断发布
```
